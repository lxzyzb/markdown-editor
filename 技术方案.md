> 编写日期：2026-06-12

---

## 一、项目背景与目标

### 1.1 项目背景
随着开发者和办公人员对轻量化、可定制、跨平台文本编辑工具的需求增长，传统的记事本类工具功能单一，而大型 IDE 体积庞大且启动缓慢。本项目旨在基于 React + Electron 框架，研发一款**轻量级、跨平台、可扩展**的文件编辑器应用，兼顾日常文本编辑与轻量代码编辑场景。

### 1.2 项目目标
- 跨平台支持：Windows / macOS / Linux
- 启动迅速、体积轻量
- 支持多文件、多标签编辑
- 提供友好的 UI 与完善的基础编辑能力
- 易于扩展（插件、主题、语法高亮等）

## 三、整体架构

### 3.1 架构分层
```
┌────────────────────────────────────────┐
│            渲染进程 (Renderer)         │
│  React UI  +  Monaco/CodeMirror 编辑器 │
└────────────────────────────────────────┘
                ↑   IPC (preload)   ↓
┌────────────────────────────────────────┐
│             主进程 (Main)              │
│  窗口管理 / 文件系统 / 菜单 / 快捷键   │
└────────────────────────────────────────┘
                ↑               ↓
┌────────────────────────────────────────┐
│       Node.js / 操作系统能力           │
│  fs、dialog、shell、autoUpdater 等     │
└────────────────────────────────────────┘
```

### 3.2 进程通信
- 使用 `contextBridge` + `ipcRenderer.invoke / ipcMain.handle` 进行安全通信
- 渲染进程不直接访问 Node API，全部通过 preload 暴露的受限 API 调用
- 数据流：UI 操作 → preload → 主进程 → 文件系统 / 系统 API → 返回结果

### 3.3 目录结构建议
```
fileEdit/
├─ electron/
│  ├─ main.ts                 # 主进程入口
│  ├─ preload.ts              # 预加载脚本
│  ├─ ipc/                    # IPC 通信模块
│  ├─ menu/                   # 应用菜单
│  └─ utils/                  # 主进程工具
├─ src/                       # 渲染进程（React）
│  ├─ main.tsx
│  ├─ App.tsx
│  ├─ components/             # 通用组件
│  ├─ views/                  # 页面视图
│  ├─ editor/                 # 编辑器封装
│  ├─ store/                  # 状态管理
│  ├─ hooks/                  # 自定义 Hooks
│  ├─ utils/                  # 工具函数
│  └─ styles/                 # 样式
├─ resources/                 # 静态资源（图标等）
├─ package.json
├─ vite.config.ts
└─ electron-builder.yml
```

## 四、核心功能模块

### 4.1 文件管理
- 新建文件（Ctrl+N）
- 打开文件（Ctrl+O），支持多选
- 保存（Ctrl+S） / 另存为（Ctrl+Shift+S）
- 关闭文件 / 关闭全部
- 最近文件列表（最多 20 条，持久化到本地）
- 文件拖拽打开（从资源管理器拖入窗口）
- 文件夹/工作区打开（项目模式）
- 关闭未保存提示
- 编码检测与切换（UTF-8、GBK 等）
- 换行符切换（LF / CRLF）
- 文件变更监听（外部修改时提示重新加载）

### 4.2 编辑能力
- 撤销 / 重做（Ctrl+Z / Ctrl+Y）
- 剪切 / 复制 / 粘贴
- 全选 / 多光标编辑
- 查找（Ctrl+F） / 替换（Ctrl+H）
- 跳转到指定行（Ctrl+G）
- 自动缩进、智能缩进
- 括号匹配、自动闭合
- 代码折叠
- 软换行（Word Wrap）开关
- 显示/隐藏不可见字符（空格、制表符、换行符）

### 4.3 多标签 / 多窗口
- 多标签页编辑，支持拖拽排序
- 标签页右键菜单：关闭、关闭其他、复制文件路径、重命名
- 支持多窗口（同一应用可打开多个独立窗口）
- 工作区状态恢复（重启后恢复上次打开的文件与布局）

### 4.4 文件浏览侧边栏
- 树形目录展示（基于打开的工作区）
- 文件/文件夹新建、删除、重命名
- 文件搜索（按名称模糊匹配）
- 显示/隐藏侧边栏
- 排除规则（node_modules、.git 等）

### 4.5 编辑器视图
- 基于 Monaco / CodeMirror 6 的核心编辑区
- 多语言语法高亮（默认支持常见语言，可扩展）
- 多主题（亮色 / 暗色 / 跟随系统）
- 字体、字号、行高、Tab 宽度可配置
- 缩略图 / 滚动条地图（Mini Map）
- 状态栏：光标位置、字符数、编码、换行符类型

### 4.6 搜索与替换
- 当前文件查找 / 替换
- 全局搜索（在打开的工作区内跨文件搜索）
- 大小写敏感、全词匹配、正则表达式
- 搜索结果高亮与快速跳转

### 4.7 Markdown 支持
- Markdown 文件语法高亮
- Markdown 实时预览（分屏模式）
- 常用快捷输入（标题、列表、代码块、链接等）

### 4.8 设置中心
- 通用设置：启动行为、默认编码、默认换行
- 编辑器设置：字体、字号、Tab 宽度、主题、自动保存
- 快捷键自定义
- 主题切换
- 设置项导入 / 导出（JSON）
- 设置持久化（本地配置文件）

### 4.9 命令面板
- 顶部命令面板（Ctrl+Shift+P）
- 模糊搜索命令并执行
- 常用操作可视化检索

### 4.10 自动更新
- 基于 `electron-updater` 实现
- 检测更新、下载、提示安装
- 支持热更新与全量更新

### 4.11 国际化（i18n）
- 基于 `react-i18next`
- 默认中文、英文，可扩展

### 4.12 快捷键体系
- 全局快捷键
- 编辑器内置快捷键
- 用户自定义快捷键（设置中心可配置）

## 六、关键技术实现

### 6.1 编辑器选型
| 方案 | 优点 | 缺点 |
|------|------|------|
| Monaco Editor | VSCode 同款，功能强大 | 体积大、加载慢 |
| CodeMirror 6 | 模块化、体积小、性能好 | 需要自行组合功能 |

**推荐**：初期采用 **Monaco** 快速验证功能；性能与体积敏感场景可切换为 **CodeMirror 6**。

### 6.2 主进程与渲染进程通信
```ts
// preload.ts
import { contextBridge, ipcRenderer } from 'electron';

contextBridge.exposeInMainWorld('api', {
  openFile: () => ipcRenderer.invoke('file:open'),
  saveFile: (path: string, content: string) =>
    ipcRenderer.invoke('file:save', { path, content }),
});
```

### 6.3 性能优化
- 编辑器懒加载（按需引入语言包）
- 大文件分块加载与按需渲染
- 状态管理精细化，避免无关组件重渲染
- 使用 Web Worker 处理搜索、语法解析等耗时操作
- 虚拟滚动（文件树、搜索结果列表）

### 6.4 安全策略
- 启用 `contextIsolation: true`
- 关闭 `nodeIntegration`
- 渲染进程通过 `contextBridge` 访问受限 API
- 文件路径校验，防止越权访问
- CSP（内容安全策略）配置

### 6.5 自动保存策略
- 失焦自动保存（可关闭）
- 定时自动保存（默认 30s，可配置）
- 保存时记录哈希，避免无变更写入

## 七、数据持久化

| 数据 | 存储方式 |
|------|----------|
| 用户设置 | `app.getPath('userData')/config.json` |
| 最近文件 | `recent.json` |
| 工作区状态 | `workspace.json` |
| 主题与扩展 | `plugins/` 目录 |
| 缓存 | `Cache` 目录（可清理） |

可使用 `electron-store` 简化本地持久化逻辑。

## 八、构建与发布

### 8.1 开发流程
```bash
# 安装依赖
pnpm install

# 启动开发（主进程 + 渲染进程热更新）
pnpm dev

# 打包
pnpm build

# 构建安装包
pnpm dist
```

### 8.2 打包配置
- 使用 `electron-builder`
- 输出格式：
  - Windows：`nsis`（安装包）、`portable`（免安装）
  - macOS：`dmg`、`zip`
  - Linux：`AppImage`、`deb`
- 自动更新服务器（可选）：GitHub Releases / 自建服务

### 8.3 版本管理
- 遵循语义化版本（SemVer）
- 主版本：重大功能变更
- 次版本：新功能
- 修订号：Bug 修复

## 十、风险与挑战

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Electron 包体积大 | 用户下载体验差 | 按需打包、压缩、tree-shaking |
| 大文件卡顿 | 编辑体验差 | 虚拟滚动、分块加载、Worker |
| 主进程崩溃 | 数据丢失风险 | 自动保存、崩溃恢复、备份机制 |
| 多平台兼容性 | 测试与维护成本 | 抽象平台差异层、CI 多平台测试 |
| 安全性 | 恶意文件利用 | CSP、contextIsolation、输入校验 |
